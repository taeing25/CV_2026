## 1. 문제 정의

### 🤔 이미지를 변환한다는 게 뭔가요?

이미지는 픽셀(점)들의 집합입니다.  
각 픽셀은 `(x, y)` 좌표를 가지고 있는데,  
**이미지 변환**이란 각 픽셀의 좌표를 새로운 좌표로 옮기는 것입니다.

```
원본 픽셀 위치       변환 후 픽셀 위치
  (100, 50)    ──▶    (180, 30)
  (101, 50)    ──▶    (181, 29)
  (102, 50)    ──▶    (182, 28)
       ...                ...
```

### 🎯 이 프로그램이 하는 일

입력 이미지(장미 사진)에 아래 3가지 변환을 **동시에** 적용합니다:

| 변환 종류 | 설정값 | 의미 |
|-----------|--------|------|
| 회전 | +30도 | 시계 반대 방향으로 30도 회전 |
| 크기 조절 | 0.8배 | 원본보다 20% 작게 축소 |
| 평행이동 | x +80px, y -40px | 오른쪽으로 80, 위로 40픽셀 이동 |

## 2. 해결 방법

### 🔑 핵심 개념: 아핀 변환(Affine Transformation)이란?

아핀 변환은 이미지의 픽셀 좌표를 **행렬 곱셈** 하나로 변환하는 방법입니다.  
회전, 크기 조절, 이동, 기울기를 모두 하나의 행렬로 표현할 수 있습니다.

```
수식으로 표현하면:

[ x' ]   [ a  b  tx ] [ x ]
[ y' ] = [ c  d  ty ] [ y ]
                      [ 1 ]

x', y' : 변환 후 좌표
x,  y  : 원본 좌표
a, b, c, d : 회전/크기 성분
tx, ty     : 이동(Translation) 성분
```

> 💡 핵심: 픽셀 하나하나를 이 행렬로 계산해서 새 위치로 옮깁니다.

---

### 📐 회전 + 크기 변환 행렬 구조

`cv2.getRotationMatrix2D(center, angle, scale)` 가 만들어주는 행렬:

```
M = [ α   β   (1-α)·cx - β·cy  ]
    [-β   α    β·cx + (1-α)·cy ]

α = scale × cos(angle)
β = scale × sin(angle)
cx, cy = 회전 중심 좌표
```

수식이 복잡해 보이지만, OpenCV가 자동으로 계산해줍니다.  
우리는 **중심점, 각도, 배율** 세 가지만 넣으면 됩니다.

---

### 🔄 평행이동을 추가하는 방법

회전 행렬에서 이동 성분은 오른쪽 열에 있습니다:

```
M = [ α   β  | tx ]   ← M[0, 2] 가 x축 이동값
    [-β   α  | ty ]   ← M[1, 2] 가 y축 이동값

M[0, 2] += 80   → x방향으로 80px 추가 이동 (오른쪽)
M[1, 2] -= 40   → y방향으로 40px 위로 이동 (y축은 아래가 +이므로 빼야 위로 감)
```

> 💡 이미지 좌표계에서 y축은 아래로 갈수록 커집니다.  
> 그래서 **위로 이동**하려면 y값을 **빼야** 합니다.

---

### 🗺️ 전체 처리 흐름

```
[이미지 읽기]
      ↓
[이미지 크기 / 중심 계산]
      ↓
[회전 + 크기 변환 행렬 생성]  ← getRotationMatrix2D()
      ↓
[평행이동 값 추가]  ← M[0,2] += 80 / M[1,2] -= 40
      ↓
[아핀 변환 적용]  ← warpAffine()
      ↓
[원본 + 결과 화면 출력 및 저장]
```

## 3. 코드 상세 설명

### 📂 Step 1. 이미지 읽기 및 기본 정보 추출

```python
img = cv2.imread('L02.image_Formation/rose.png')

h, w = img.shape[:2]
center = (w / 2, h / 2)
```

| 변수 | 의미 |
|------|------|
| `h, w` | 이미지의 높이(행 수)와 너비(열 수) |
| `center` | 회전의 기준이 될 이미지 정중앙 좌표 |
| `img.shape` | `(높이, 너비, 채널 수)` 형태의 튜플, `[:2]`로 높이·너비만 추출 |

> ⚠️ `img.shape`은 `(높이, 너비)` 순서입니다.  
> 하지만 좌표는 `(x, y)` = `(너비 방향, 높이 방향)` 순서입니다.  
> 헷갈리기 쉬운 부분이니 주의하세요!

---

### 🔄 Step 2. 회전 + 크기 변환 행렬 생성

```python
M = cv2.getRotationMatrix2D(center, 30, 0.8)
```

| 인자 | 값 | 의미 |
|------|-----|------|
| `center` | `(w/2, h/2)` | 회전의 중심점 (이미지 정중앙) |
| `30` | 30도 | 회전 각도 (양수 = 반시계 방향) |
| `0.8` | 0.8배 | 크기 배율 (1.0이면 크기 유지) |

결과로 만들어지는 `M`의 형태 (예시):

```
M = [[ 0.693  0.400   tx ],
     [-0.400  0.693   ty ]]

(2행 3열 행렬)
```

---

### ➡️ Step 3. 평행이동 추가

```python
M[0, 2] += 80   # x축 오른쪽으로 80픽셀 이동
M[1, 2] -= 40   # y축 위쪽으로 40픽셀 이동
```

행렬의 각 위치가 의미하는 것:

```
M = [[ a    b   | M[0,2] ]]  ← x 이동값  (열 인덱스 2)
    [[ c    d   | M[1,2] ]]  ← y 이동값  (행 인덱스 1)
```

회전만 하고 싶으면 이 부분을 건드리지 않아도 됩니다.  
여기서는 회전 후 추가로 위치를 이동시키기 위해 값을 더하고 뺍니다.

---

### 🖼️ Step 4. 아핀 변환 적용

```python
dst = cv2.warpAffine(img, M, (w, h))
```

| 인자 | 의미 |
|------|------|
| `img` | 변환할 원본 이미지 |
| `M` | 앞서 만든 2×3 변환 행렬 |
| `(w, h)` | 출력 이미지 크기 (원본과 동일하게 유지) |

> 💡 변환된 픽셀이 출력 범위를 벗어나면 잘립니다.  
> 출력 크기를 `(w*2, h*2)` 처럼 크게 잡으면 잘림 없이 전체를 볼 수 있습니다.

---

### 🖥️ Step 5. 결과 출력 및 저장

```python
cv2.imshow('Original', img)      # 원본 이미지 창
cv2.imshow('Transformed', dst)   # 변환된 이미지 창
cv2.waitKey(0)                   # 아무 키나 누를 때까지 대기
cv2.destroyAllWindows()          # 모든 창 닫기

combined = cv2.hconcat([img, dst])  # 두 이미지를 가로로 나란히 붙임
cv2.imwrite('rose.jpg', combined)   # 파일로 저장
```

`cv2.hconcat()` 동작 방식:

```
[ 원본 이미지 ] + [ 변환 이미지 ]  →  [ 원본 | 변환 ]
   (w × h)          (w × h)             (2w × h)
```

## 4. 주의사항 및 버그 수정

### ⚠️ 버그: 들여쓰기 오류

원본 코드 마지막 부분에 불필요한 들여쓰기가 있습니다:

```python
# ❌ 잘못된 코드
combined = cv2.hconcat([img, dst])

    # - 원본과 흑백이 나란히 붙은 사진 저장  ← 들여쓰기 오류! IndentationError 발생
cv2.imwrite('rose.jpg', combined)

# ✅ 올바른 코드
combined = cv2.hconcat([img, dst])
# 원본과 변환 이미지가 나란히 붙은 사진 저장
cv2.imwrite('rose.jpg', combined)
```

---

### ⚠️ 주의: 이미지 좌표계의 y축 방향

```
일반 수학 좌표계        이미지 좌표계
      ↑ y+             (0,0) ──▶ x+
      │                  │
      │                  ↓ y+
──────┼──▶ x+
```

이미지에서 y축은 **아래로 갈수록 증가**합니다.  
그래서 `M[1, 2] -= 40`이 **위쪽** 이동입니다.

---

### ⚠️ 주의: 회전 각도 방향

```python
cv2.getRotationMatrix2D(center, 30, 0.8)
#                                ↑
#                         양수 = 반시계 방향
#                         음수 = 시계 방향
```

### 💡 파라미터 변경 실험해보기

| 바꿔볼 값 | 효과 |
|-----------|------|
| `30` → `90` | 90도 회전 |
| `0.8` → `1.5` | 1.5배 확대 |
| `+= 80` → `+= 0` | 좌우 이동 없음 |
| `(w, h)` → `(w*2, h*2)` | 출력 캔버스를 크게 해서 잘림 방지 |
