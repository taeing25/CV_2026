## 1. 문제 정의

### 🤔 카메라로 찍은 사진이 실제와 다른 이유

카메라는 3D 공간을 2D 이미지로 담습니다.  
이 과정에서 두 가지 문제가 생깁니다.

**① 렌즈 왜곡 (Distortion)**

카메라 렌즈, 특히 광각 렌즈는 빛을 굴절시키면서 이미지를 휘게 만듭니다.  
직선이 곡선처럼 보이는 현상이 생깁니다.

```
실제 모습 (직선)       카메라로 찍은 모습 (왜곡)
┌──────────────┐       ┌──────────────┐
│              │       │  (  휘어짐  ) │
│              │  ──▶  │ (           )│
│              │       │  (         ) │
└──────────────┘       └──────────────┘
```

**② 카메라 내부 파라미터를 모름**

픽셀 좌표를 실제 3D 좌표로 바꾸려면 카메라의 **초점 거리(f)** 와 **이미지 중심(cx, cy)** 을 알아야 합니다.  
이 값들은 카메라마다 다르고, 카탈로그에 정확히 나와있지 않습니다.

---

### 🎯 이 프로그램의 목표

- 체크보드 이미지 여러 장을 이용해 카메라의 **내부 파라미터 K** 를 자동으로 계산한다
- **왜곡 계수(dist)** 를 구해서 찍힌 이미지의 왜곡을 제거한다
- 보정 전/후 이미지를 나란히 저장해 결과를 비교한다

---

## 2. 해결 방법

### 🔑 핵심 아이디어: 체크보드를 기준점으로 활용

체크보드는 코너(격자 교차점)의 **실제 3D 위치를 정확히 알 수 있는** 패턴입니다.

```
실제 체크보드 (3D)         이미지에서 검출된 코너 (2D)

(0,0)  (25,0)  (50,0)          (132, 87)  (165, 88) ...
  ●──────●──────●                  ●──────────●
  │      │      │       ──▶        │          │
  ●──────●──────●                  ●──────────●
(0,25) (25,25) (50,25)          (131,121) (164,122) ...

"실제 위치를 알고 있는 점들"이   "이미지에서 어디 찍혔는지"
각각 이미지의 어디에 찍혔는지 비교 → 카메라 파라미터 역산
```

이 대응 관계를 여러 장(다양한 각도)으로 수집해서 수학적으로 풀면 카메라 파라미터가 나옵니다.

---

### 📐 카메라 행렬 K란?

```
K = [ fx   0   cx ]
    [  0  fy   cy ]
    [  0   0    1 ]

fx, fy : x축, y축 초점 거리 (픽셀 단위)
cx, cy : 이미지 중심 좌표 (주점, Principal Point)
```

이 행렬이 있으면 **픽셀 좌표 ↔ 실제 3D 좌표** 변환이 가능해집니다.

---

### 🔄 왜곡 보정 원리

```
왜곡된 이미지 픽셀
      ↓
왜곡 계수(dist)로 각 픽셀의 원래 위치 계산
      ↓
픽셀들을 원래 위치로 재배치
      ↓
보정된 이미지 완성
```

---

### 🗺️ 전체 처리 흐름

```
[체크보드 이미지 여러 장 읽기]
          ↓
[각 이미지에서 코너 검출]  ← findChessboardCorners()
          ↓
[코너 위치를 서브픽셀 정밀도로 정제]  ← cornerSubPix()
          ↓
[실제 3D 좌표 ↔ 이미지 2D 좌표 대응 수집]
          ↓
[카메라 캘리브레이션 실행]  ← calibrateCamera()
    → 카메라 행렬 K, 왜곡 계수 dist 출력
          ↓
[왜곡 보정 적용]  ← undistort()
          ↓
[보정 전/후 이미지 저장]
```

## 3. 코드 상세 설명

### 📂 Step 0. 기본 설정

```python
CHECKERBOARD = (9, 6)   # 체크보드의 내부 코너 개수 (가로 9개, 세로 6개)
square_size = 25.0      # 체크보드 한 칸의 실제 크기 (25mm)
```

> ⚠️ `CHECKERBOARD`는 체크보드 **전체 칸 수**가 아니라 **내부 코너 개수**입니다.  
> 10×7 칸짜리 체크보드라면 내부 코너는 `(9, 6)`이 됩니다.

```python
criteria = (cv2.TERM_CRITERIA_EPS + cv2.TERM_CRITERIA_MAX_ITER, 30, 0.001)
```

코너 정밀화 반복을 언제 멈출지 결정하는 조건입니다:

| 항목 | 값 | 의미 |
|------|-----|------|
| `MAX_ITER` | 30 | 최대 30번 반복 |
| `EPS` | 0.001 | 변화량이 0.001 이하면 조기 종료 |

---

### 📐 Step 1. 실제 3D 좌표 생성

```python
objp = np.zeros((CHECKERBOARD[0]*CHECKERBOARD[1], 3), np.float32)
objp[:, :2] = np.mgrid[0:CHECKERBOARD[0], 0:CHECKERBOARD[1]].T.reshape(-1, 2)
objp *= square_size
```

체크보드가 평평하게 놓여있다고 가정하기 때문에 **z좌표는 항상 0**입니다.

```
생성되는 objp의 모습:

[[ 0.0,  0.0,  0.0],   ← 첫 번째 코너 (0mm, 0mm, 0mm)
 [25.0,  0.0,  0.0],   ← 두 번째 코너 (25mm, 0mm, 0mm)
 [50.0,  0.0,  0.0],   ← 세 번째 코너
 ...
 [200.0, 125.0, 0.0]]  ← 마지막 코너 (9×25mm, 6×25mm, 0mm)
```

---

### 🔍 Step 2. 체크보드 코너 검출

```python
ret, corners = cv2.findChessboardCorners(gray, CHECKERBOARD, None)
```

| 반환값 | 의미 |
|--------|------|
| `ret` | 코너 검출 성공 여부 (`True` / `False`) |
| `corners` | 검출된 코너의 2D 이미지 좌표 배열 |

```python
corners2 = cv2.cornerSubPix(gray, corners, (11, 11), (-1, -1), criteria)
```

`findChessboardCorners()`는 정수 픽셀 단위로 코너를 찾습니다.  
`cornerSubPix()`는 이를 **소수점 픽셀 단위**(서브픽셀)로 정밀하게 보정해줍니다.  
캘리브레이션 정확도가 크게 향상됩니다.

```
정밀화 전: 코너 위치 (132, 87)
정밀화 후: 코너 위치 (132.47, 87.23)  ← 훨씬 정밀
```

---

### 📸 Step 3. 코너 시각화

```python
cv2.drawChessboardCorners(img, CHECKERBOARD, corners2, ret)
cv2.imshow('img', img)
cv2.waitKey(500)   # 0.5초 동안 보여주고 다음 이미지로 넘어감
```

각 이미지에서 검출된 코너를 확인하며 진행 상황을 시각적으로 확인할 수 있습니다.

---

### 🎯 Step 4. 카메라 캘리브레이션

```python
ret, K, dist, rvecs, tvecs = cv2.calibrateCamera(
    objpoints,   # 실제 3D 좌표 목록
    imgpoints,   # 이미지 2D 좌표 목록
    img_size,    # 이미지 크기 (w, h)
    None, None   # 초기값 없음 (자동 추정)
)
```

| 반환값 | 의미 |
|--------|------|
| `ret` | 재투영 오차 (낮을수록 정확, 보통 1.0 이하면 좋음) |
| `K` | 카메라 내부 행렬 (3×3) |
| `dist` | 왜곡 계수 (k1, k2, p1, p2, k3) |
| `rvecs` | 각 이미지에서의 카메라 회전 벡터 |
| `tvecs` | 각 이미지에서의 카메라 이동 벡터 |

---

### 🔧 Step 5. 왜곡 보정

```python
newcameramtx, roi = cv2.getOptimalNewCameraMatrix(K, dist, (w, h), 1, (w, h))
dst = cv2.undistort(img, K, dist, None, newcameramtx)
```

**왜 `getOptimalNewCameraMatrix()`를 먼저 쓰나요?**

왜곡을 보정하면 이미지 가장자리에 **검은 빈 영역**이 생길 수 있습니다.  
`getOptimalNewCameraMatrix()`는 이 빈 영역을 얼마나 허용할지 조정한 새 카메라 행렬을 만들어줍니다.

```
alpha=1 (기본값): 빈 영역 포함, 원본 픽셀 최대한 보존
alpha=0:          빈 영역 제거, 유효한 픽셀만 남김
```

---

### 💾 Step 6. 결과 저장 및 출력

```python
combined = cv2.hconcat([img, dst])
cv2.imwrite('calibration.jpg', combined)

cv2.imshow('Original', img)
cv2.imshow('Undistorted', dst)
cv2.waitKey(0)
```

보정 전/후를 나란히 붙여서 한눈에 비교할 수 있게 저장합니다.

```
[ 원본 (왜곡됨) | 보정된 이미지 ]
```

## 4. 주의사항
---

### ⚠️ 주의: 코너 개수를 정확히 입력해야 함

```python
CHECKERBOARD = (9, 6)
```

체크보드의 **내부 코너** 개수입니다. 칸 수와 헷갈리지 마세요.

```
10칸 × 7칸 체크보드
┌──┬──┬──┬──┬──┬──┬──┬──┬──┬──┐
│  │  │  │  │  │  │  │  │  │  │   ← 10칸
├──┼──┼──┼──┼──┼──┼──┼──┼──┼──┤
│  │  │  │  │  │  │  │  │  │  │
...
└──┴──┴──┴──┴──┴──┴──┴──┴──┴──┘
                                     7칸

내부 코너 수 = (10-1) × (7-1) = 9 × 6  ✅
```

---

### ⚠️ 주의: 이미지가 충분해야 정확함

캘리브레이션의 정확도는 **이미지 장 수와 다양성**에 크게 영향받습니다.

| 조건 | 권장사항 |
|------|----------|
| 최소 이미지 수 | 10장 이상 |
| 체크보드 각도 | 다양한 기울기 포함 |
| 체크보드 위치 | 이미지 가장자리까지 골고루 |
| 재투영 오차 | 1.0 픽셀 이하면 양호 |

---

### 💡 재투영 오차(ret)로 캘리브레이션 품질 확인

```python
print(f"재투영 오차: {ret:.4f} px")
# 0.5 이하 → 매우 좋음
# 1.0 이하 → 양호
# 1.0 이상 → 이미지 추가 또는 흔들린 이미지 제거 권장
```
